Start Development

Snap comes with a development environment that is available over HTTP on any web browser that supports Web Sockets. To develop and debug scripts the devlopment environment can be started with the command shown below. All that is required is a web browser and Java 1.5 or greater. The development environment is a single JAR file that can be downloaded from here. In addition an embeddable version is available which can be integrated in to any Java application.

java -jar snapd.jar --mode=develop --directory=work --port=4457

The development environment can use hot stand-by agents to improve responsiveness, the agent pool can be configured on the command line. In addition the agent connect port can be specified to enable Android applications to connect to the development environment. An example configuration is shown below.

java -jar snapd.jar --mode=develop --directory=work --port=4457 --agent-pool=4 --agent-port=4456
Set Breakpoints

For any reasonably sized application the ability to debug is essential. A development environment is provided that assists in debugging scripts. It has the ability to set breakpoints, and to step through code one statement at a time. During the debug process the editor highlights the line that is to be executed next.


Inspect Variables

During the debugging process it is often useful to determine the state of the application. To assist the development environment provides a variables panel that can be used to dive through the object graph that is accessible within the executing scope.


Profile Execution

To access application performance the debugger contains an internal profiler. This profiler is used to track the duration of time the application spends in any one part of the application. Information regarding the profiling process is updated so that statistics can be compiled and viewed.


View Console

As an application is executing its console output is intercepted and routed to the development environment console tab. This allows a program output to be viewed as it executes.
